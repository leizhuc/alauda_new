# Jira Insight Bot - 架构设计文档

## 1. 项目简介
Jira Insight Bot 是一个为产品经理量身定制的本地化 AI 辅助工具。它通过每天定时或手动拉取内网 Jira 中最新解决的工单评论，利用大模型自动分析根本原因（Root Cause），过滤无效信息，并提炼出需要产品经理关注的产品改进建议。

## 2. 核心架构选型
本项目采用完全本地运行的轻量级 Web 架构，确保内网数据安全，并提供直观的可视化界面。

*   **前端 UI / 交互层:** `Streamlit` (纯 Python 构建的现代数据看板 UI)
*   **大模型代理层:** `LiteLLM` (支持一套代码无缝切换主流大模型)
*   **Jira 交互层:** `jira` (Python 官方推荐库)
*   **本地存储层:** 本地文件系统 (JSON 或 SQLite) 缓存历史分析结果

## 3. 大模型支持矩阵
系统设计为多模型插拔式，通过 Streamlit 侧边栏进行切换：
*   **主力模型 (Primary):** `gemini-2.5-flash` (速度极快，适合大批量文本总结)
*   **免费/备选公网模型:** `Groq` (Llama-3 系列), `DeepSeek`
*   **纯内网/本地模型:** `Ollama` (如需极致隐私，可本地运行模型)

## 4. 核心工作流与配置设计
1.  **动态配置中心 (UI 侧边栏):** 
    *   用户输入 Jira 内网地址、账号凭证及 LLM API Key。
    *   **[关键特性] 自定义 JQL 查询:** 提供输入框，允许 PM 自定义需要关注的范围（例如：`project = 'ALAUDA' AND status changed to Resolved during (-24h, now())`）。
2.  **数据抓取:** 根据用户配置的自定义 JQL 抓取近期解决的工单及全部 Comment。
3.  **清洗与组装:** 提取工单核心字段和排查回复，组装 Prompt。
4.  **AI 分析:** LLM 判断问题分类、是否需 PM 关注，并输出改进建议。
5.  **看板展示:** UI 渲染出“需关注工单”列表，供 PM 每日晨会前快速 Review。
